# プロジェクト引き継ぎドキュメント: 書道教室Schedule

## 1. プロジェクト概要
このプロジェクトは、書道教室の運営を効率化するためのスケジュール管理システムです。管理者が授業日（開講日）を設定し、生徒を月間の回数制限（月2回/3回）に合わせて割り振る機能、および生徒自身が自分の予定を確認・変更できる機能を提供します。

## 2. 技術スタック
- **Frontend**: Next.js 15 (App Router), TypeScript, Tailwind CSS
- **UI Components**: ShadCN UI, Lucide React (Icons)
- **Backend/Database**: Firebase (Authentication, Firestore, App Hosting)
- **AI/GenAI**: Genkit (将来的な自動割り振り機能などのために導入済み)
- **Date Handling**: date-fns (日本語ロケール設定済み)

## 3. 主要機能
### 管理者側 (/admin)
- **ダッシュボード**: 各管理機能へのクイックアクセス。
- **月間スケジュール**: カレンダー形式で全スロットの予約状況を確認・編集。
- **月間割り振り (最重要)**: 「生徒プール」から「日付バケツ」へ生徒をドラッグするように割り振る機能。月間上限チェック機能付き。
- **本日の運営**: `/admin/today` にアクセスすると、今日が休みなら「次の授業日」へ自動リダイレクトする親切設計。
- **開講日設定**: 月ごとにどの日を授業日にするかをカレンダーで一括設定。
- **生徒管理**: 生徒情報のCRUD。アカウント連携用の「生徒コード(@std...)」と「6桁トークン」の自動生成。
- **振替申請管理**: 生徒からの振替申請の承認・却下。

### 生徒側 (/student)
- **ダッシュボード**: 自分の次回授業予定とお知らせの確認。
- **全授業一覧**: 過去と未来の授業リスト。未来の授業については、空きスロットへの「セルフ移動」が可能。
- **お知らせ**: 教室からの連絡事項の閲覧。
- **アカウント連携**: 管理者から渡されたコードを入力し、自分のログインアカウントと生徒データを紐付ける。

## 4. データ構造 (Firestore)
- `users/{uid}`: 認証ユーザー。`role` (admin/student) と `linkedStudentId` を保持。
- `students/{id}`: 生徒のマスターデータ。コース、連携コード、希望時間帯を保持。
- `lessons/{id}`: 個別の授業記録。`slotId` (格式: `YYYY-MM-DD-HH:mm`) で時間枠を管理。
- `announcements/{id}`: お知らせ。
- `swapRequests/{id}`: 振替申請。
- `settings/app`: アプリ全体設定。`activeDatesByMonth` (月ごとの開講日リスト) を保持。

## 5. 技術的な工夫と解決済みの課題
- **N+1問題の解決**: ロード時間を短縮するため、生徒一人ひとりの予約を個別に取得せず、`getMonthlyLessonCounts` 等で月間の全データを一括取得（Batch Loading）してメモリ上で集計する方式を採用しています。
- **インデックスエラーの回避**: Firestoreの複合インデックス作成の手間を避けるため、クエリはシンプルに保ち、複雑なフィルタリングはクライアントサイド（またはAPI内）で行っています。
- **UIフリーズ対策**: ShadCNのSheet（サイドバー）とDialogが競合して背景が固まる問題を、非同期処理による表示タイミングの調整で解決しています。
- **Admin SDKの堅牢化**: サーバーサイドでのJSONパースエラーをガードし、環境変数が未設定でもビルドが通るように設計されています。

## 6. 今後の開発課題
1. **自動割り振り機能 (AI)**: 生徒の「希望曜日・時間」に基づき、Genkitを使用して最適な月間スケジュールを1クリックで生成する機能。
2. **プッシュ通知**: 授業前日のリマインドやお知らせ投稿時の通知。
3. **入退室記録**: QRコード等を用いた、実際の出席確認機能。

## 7. 注意事項
- サーバーサイドの機能（アカウント連携やレッスン移動）を動作させるには、環境変数 `FIREBASE_SERVICE_ACCOUNT_KEY` (JSON形式) が必須です。
- 開発時は `npm run dev` でポート 9002 を使用します。
